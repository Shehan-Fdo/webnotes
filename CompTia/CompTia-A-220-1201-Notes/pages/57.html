Power-On Self-Test (POST) and Hardware Troubleshooting

CompTIA A+ 220-1201 - Core 1 | Lesson 5.1

Power-On Self-Test (POST)

  • The Power-On Self-Test (POST) is a series of diagnostic tests performed when a computer is powered on.
  • Checks core components for proper installation and functionality:
    • CPU: Verifies installation and communication with BIOS.
    • Memory (RAM): Ensures memory modules are detected and operational.
    • Video subsystem: Confirms the display adapter is working.
    • BIOS hardware: Validates basic input/output system functionality.
  • If a problem is detected, the system may display an error message or emit a series of beep codes.
  • Beep codes vary by manufacturer and motherboard; always refer to the manufacturer’s documentation for interpretation.

Common POST Errors and Troubleshooting

  • No Display (Black Screen with Beeps):
    • Possible causes: faulty video subsystem, bad RAM, or CPU issues.
    • Check connections, reseat components, or test with known-good hardware.
  • Incorrect BIOS Time/Date:
    • Indicates a failed CMOS battery on the motherboard.
    • Replace the button-style battery (e.g., CR2032) to resolve.
  • Unexpected Boot Device:
    • System may attempt to boot from a USB drive or incorrect storage device.
    • Adjust the boot order in BIOS/UEFI settings to prioritize the correct device.

Windows Stop Errors (Blue Screen of Death - BSOD)

  • A Windows stop error (BSOD) occurs when the OS encounters an unrecoverable issue.
  • Displays a stop code (e.g., 0xC0000135) and may reference a faulty driver or hardware.
  • Troubleshooting steps:
    • Note the stop code and search for solutions online.
    • Use Last Known Good Configuration or System Restore to revert recent changes.
    • Boot into Safe Mode to diagnose driver or software conflicts.
    • Check Event Viewer for detailed error logs.
    • Run hardware diagnostics to test components like RAM, CPU, and storage.

Application Errors and Troubleshooting

  • Common symptoms:
    • Application crashes or freezes.
    • Error messages with codes (e.g., Error 47829).
    • Applications disappearing without warning.
  • Troubleshooting tools:
    • Event Viewer: Logs system and application errors.
    • Reliability Monitor: Tracks application crashes and system stability over time.
    • Reinstallation: Uninstall and reinstall the application to resolve corruption.
  • Document error details, including screenshots, for manufacturer support.

Display Issues and Troubleshooting

  • Black Screen:
    • Check power and video cable connections (HDMI, DisplayPort, etc.).
    • Verify the monitor’s input source matches the connected cable.
    • Test with a known-good monitor to rule out hardware failure.
  • Dim or Unreadable Display:
    • Adjust brightness/contrast settings on the monitor.
  • Display Works During POST but Fails in Windows:
    • Boot into VGA Mode (via F8 during startup) to load a generic video driver.

Power Issues and Troubleshooting

  • No Power:
    • Check the power outlet and power cord for faults.
    • Use a multimeter to test AC (outlet) and DC (power supply) voltages.
    • Inspect the power supply unit (PSU) for failure.
  • Fans Spin but No Display:
    • Indicates a POST failure; check motherboard, CPU, or video card.
    • Test voltage outputs from the PSU with a multimeter.

Performance Issues and Troubleshooting

  • Slow Performance:
    • Check Task Manager for high CPU, memory, or disk usage.
    • Update the OS and drivers via Windows Update.
    • Free up disk space and defragment (if using HDD).
    • Disable power-saving modes that throttle CPU performance.
    • Run anti-virus/anti-malware scans to detect malicious software.

Overheating and Cooling Issues

  • Heat sources in a computer:
    • CPU, GPU, and memory modules generate significant heat.
  • Cooling methods:
    • Fans: Pull in cool air and expel hot air.
    • Heat sinks: Dissipate heat from components.
    • Ensure fans are clean and unobstructed to maintain airflow.
  • Monitor temperatures using software like HWMonitor.
  • Symptoms of Overheating:
    • System randomly shuts down to prevent damage.
    • Check Event Viewer for temperature-related logs.
    • Inspect thermal paste on CPU/GPU and reapply if necessary.

Hardware Failures and Physical Damage

  • Burning Smell or Smoke:
    • Immediately power off the system to prevent further damage.
    • Inspect the motherboard and components for visible damage.
  • Unusual Noises:
    • Grinding: Likely a failing hard drive.
    • Rattling: Loose components (e.g., heat sink).
    • Clicking: Obstruction in a fan or failing hardware.
    • Popping: Blown capacitor on the motherboard.
  • Capacitor Failure:
    • Look for bulging or leaking capacitors on the motherboard.
    • Replace the motherboard or affected component if damaged.

BIOS/UEFI Configuration and Maintenance

  • CMOS Battery Replacement:
    • Replace the button battery (e.g., CR2032) to fix date/time resets.
    • Modern systems retain BIOS settings even after battery removal.
  • Resetting BIOS Settings:
    • Use a jumper on the motherboard to reset BIOS to defaults.
    • Removing the CMOS battery does not always reset passwords or settings.
